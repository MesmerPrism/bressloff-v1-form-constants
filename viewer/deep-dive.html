Deep dive

From cortical symmetry to flicker-driven form constants

This page is the notebook-style companion to the interactive lab. It explains why the Bressloff orientation-hypercolumn model and the Rule, Stoffregen, and Ermentrout flicker model are separate model families, then walks through generated examples one at a time.

The animations are generated implementation output. They are not scans of the source papers, not clinical predictions, and not calibrated reconstructions of any individual visual experience.

The historical problem

Geometric visual hallucinations are recurring reports with a stubborn mathematical shape: tunnels, spirals, rays, cobwebs, lattices, honeycombs, checkerboards, and funnels. The modeling question is not merely whether those pictures can be drawn. It is whether a plausible cortical system has natural pattern-forming modes that look like them.

Ermentrout and Cowan had already shown how a two-dimensional cortical sheet with Mexican-hat interactions can destabilize into stripes, squares, and hexagons. Bressloff, Cowan, Golubitsky, Thomas, and Wiener add two crucial pieces: the retinocortical map between visual space and V1, and the orientation preference structure of V1 hypercolumns. That upgrade turns some hallucinations from simple light-dark regions into fields of local contour elements.

Scalar cortical sheet

Useful for rings, rays, spirals, and some light-dark lattices because cortical stripes map cleanly through the complex-log retinocortical transform.

Orientation hypercolumns

Needed for cobwebs, honeycombs, and lattice tunnels that are more naturally described as locally oriented contours rather than only bright and dark regions.

The Bressloff model family

The Bressloff track in this repository is an orientation-hypercolumn model family: 

model_family = bressloff_orientation_hypercolumn

Its state is not just activity over cortical position. It is activity over cortical position and orientation preference. That extra orientation coordinate is what lets the renderer draw local contour elements, distinguish even and odd branches, and apply the double retinocortical map used by the paper's contoured examples.

Position map

z = x + i y
z ~= log(r_R exp(i theta_R))
x ~= log r_R
y ~= theta_R

Circles, rays, and logarithmic spirals in visual space become vertical, horizontal, and oblique stripes in cortical coordinates.

Orientation map

phi = phi_R - theta_R
equivalently:
phi_R = phi + theta_R

Local retinal contour orientation is measured relative to angular visual-field position. The implementation uses the equivalent phi_R = phi + theta_R convention for drawing glyphs.

Hypercolumn field

da(r, phi, t)/dt =
  -a(r, phi, t)
  + mu integral w sigma(a)

The weight separates local isotropic coupling inside a hypercolumn from anisotropic long-range lateral coupling between similarly tuned orientation patches.

Planform modes

a(r, phi) ~
  sum_j c_j u(phi - phi_j)
        exp(i k_j dot r)

Near instability, the candidate patterns are finite combinations of orientation-modulated cortical plane waves on square, rhombic, or hexagonal lattices.

What the code computes

The viewer separates rendering from mathematical status. A preset can explicitly render a source-targeted figure while the current normalized stability calculation still reports a different branch. That distinction is deliberate: it keeps the public page from overstating calibration.

  1. Choose a model family. Bressloff uses orientation-channel planforms and dynamics. Rule uses a scalar excitatory/inhibitory flicker field.
  2. Generate cortical activity. Bressloff either solves dynamics or renders analytic planforms. Rule integrates a flicker-driven E/I field.
  3. Analyze status. Bressloff reports contour mode, parity, rendered family, and branch-selection diagnostics. Rule reports spatial family and temporal response mode.
  4. Map to the viewing surface. Scalar activity can be shown directly or sampled through the retinocortical map. Bressloff contoured planforms also draw local orientation glyphs through the double map.

The Rule flicker model is a separate family

Rule, Stoffregen, and Ermentrout ask a different first question: what patterns does a periodically forced excitatory-inhibitory field produce under diffuse flicker? This is not an orientation-hypercolumn preset: 

model_family = rule_flicker_ei
generator = rule_flicker

E/I field

tau_e dUe/dt =
  -Ue + F(aee Ke*Ue - aei Ki*Ui - theta_e + ge S)

tau_i dUi/dt =
  -Ui + F(aie Ke*Ue - aii Ki*Ui - theta_i + gi S)

The implementation uses two scalar fields over a periodic 2D domain with normalized Gaussian spatial kernels.

Flicker stimulus

S(t) = A H(sin(2 pi t / T) - threshold)

Period T, amplitude A, threshold, smoothing, and feed-forward inhibitory fraction are Rule-specific controls.

Temporal check

high frequency: response period ~= 2T
low frequency:  response period ~= T

The qualitative report compares the final frames against frames one and two stimulus periods earlier.

Deferred stability

homogeneous periodic orbit
linearized mode equation
monodromy matrix
Floquet multipliers

These are the next rigorous layer for Rule Figures 3, 6, 8, and 9. The page below is still simulation-first.

What remains open

The next mathematical work is calibration rather than adding more panels. Bressloff needs source-fitted phase conventions, digitized curve and image metrics, and a higher-order odd-hexagonal selector. Rule needs frequency-amplitude sweeps, Floquet stability around the homogeneous periodic orbit, and exact figure-level parameter checks.

The next writing work is to turn this page from a strong guided notebook into a source-annotated essay: add compact citations near each formula, add one still or report panel for stability curves, and split the 2002 Figure 6 representative into per-subpanel examples once the source geometry is calibrated.
